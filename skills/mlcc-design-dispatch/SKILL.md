# MLCC 설계 검증 및 적층투입지시

DOE/신뢰성 시뮬레이션에서 확정된 최종 설계값을 실제 공정에 적용하기 위한 3단계 검증 및 투입 워크플로우를 오케스트레이션한다.

## 실행 원칙

- tool 호출이 실패하거나 결과가 0건이어도 즉시 포기하지 않는다. 검색 조건을 완화(tolerance 증가)하거나, 다른 파라미터로 재시도하거나, 사용자에게 대안을 제시한다.
- 사용자가 "다시 해봐", "다른 방법으로", "범위 바꿔서" 등 재시도를 요청하면 반드시 응한다. 이미 시도한 조건을 변경하여 최소 2~3회는 다른 접근을 시도한다.
- "안됩니다"로 끝내지 말고, 항상 다음에 시도할 수 있는 대안(tolerance 확대, 부분 매칭, 다른 검색 조건 등)을 함께 제시한다.

## Reference 문서

각 단계를 실행할 때 해당 reference를 반드시 참고한다.

- `references/screen-plate-search.md`: 스크린 동판 검색 가이드. **1단계 실행 전에 반드시 참고한다.**
- `references/running-chips-search.md`: 실제 칩 정보 검색 가이드. **2단계 실행 전에 반드시 참고한다.**

## 3단계 워크플로우

이 skill은 다음 3단계를 순서대로 진행한다. 각 단계는 독립적으로 실행할 수도 있고, "설계값 검증하고 투입까지 해줘" 같은 요청에는 3단계를 연속 실행한다.

### 1단계: 스크린 동판 존재 확인

최종 설계값의 screen 관련 치수(길이, 너비, 마진)를 충족하는 스크린 동판이 현재 공정에 실제 존재하는지 확인한다.

- Tool: `search_screen_plate(screen_chip_size_leng, screen_chip_size_widh, ...)`
- 매칭 결과가 있으면 → 동판 정보를 사용자에게 제시
- 매칭 결과가 없으면 → "스크린 동판 신규 제작이 필요합니다" 안내
- 상세 가이드: `references/screen-plate-search.md`

### 2단계: 실제 칩 정보 검색

해당 설계값과 유사한 조건으로 현재 공정에서 실제로 흐르고 있는 칩들의 정보를 검색한다. 생산 실적 확인 및 투입 전 참고 용도이다.

- Tool: `search_running_chips(chip_prod_id, active_layer, cast_dsgn_thk, ...)`
- chip_prod_id 패턴 매칭 또는 설계 파라미터 범위 매칭
- 상세 가이드: `references/running-chips-search.md`

### 3단계: 적층투입지시

최종 설계값으로 적층 공정에 투입 지시를 실행한다. **이 단계는 실제 생산 라인에 영향을 주므로 반드시 사용자 최종 확인을 거쳐야 한다.**

- Tool: `dispatch_stacking_order(chip_prod_id, lot_id, design_values, user_confirmed)`
- 첫 호출 시 `user_confirmed=False` → 설계값 요약 제시 및 확인 요청
- 사용자 "예" 확인 후 → `user_confirmed=True`로 재호출하여 실제 투입 실행
- 상세 가이드: `references/dispatch-workflow.md`

## 핵심 원칙

- 이전 스킬(mlcc-optimal-design-doe)에서 도출된 최종 설계값을 입력으로 받는다. 설계값이 없으면 먼저 요청한다.
- 3단계는 순서대로 진행하는 것을 권장하지만, 사용자가 특정 단계만 요청하면 해당 단계만 실행한다.
- **투입지시(3단계)는 반드시 사용자 확인 후 실행한다.** 자동 투입은 허용하지 않는다.
- 동판 검색 또는 칩 검색 결과가 없다고 해서 투입을 차단하지는 않는다. 결과를 사용자에게 안내하고 판단을 맡긴다.
- `search_query_database`를 활용한 자유 쿼리도 가능하다. 스크린 동판이나 칩 정보의 추가 조건 검색이 필요할 때 사용한다.

## 패턴 라우팅

| 요청 유형 | 실행 단계 |
|---|---|
| "스크린 동판 있는지 확인해줘" | 1단계만 |
| "이 설계값으로 흐르는 칩 있어?" | 2단계만 |
| "적층투입지시 해줘" | 3단계만 (확인 포함) |
| "설계값 검증하고 투입해줘" | 1단계 → 2단계 → 3단계 순차 |
| "동판 확인하고 투입까지" | 1단계 → 3단계 |

## 대화 규칙

- 설계값의 핵심 수치를 항상 함께 보여준다 (screen 치수, active_layer, cast_dsgn_thk 등)
- 동판 검색/칩 검색 결과는 간결한 테이블 형태로 요약한다
- 투입지시 전에는 반드시 설계값 전체를 보여주고 확인을 받는다
- 투입 성공 후에는 dispatch_id를 포함한 완료 메시지를 전달한다

## 실패 처리

- 동판 없음 → tolerance를 늘려서 재검색한다. 여전히 없으면 "해당 치수의 스크린 동판이 없습니다. 신규 제작이 필요할 수 있습니다." + 사용자에게 진행 여부 확인. tolerance 확대, 부분 매칭 등 대안을 함께 제시한다.
- 칩 없음 → tolerance_pct를 늘리거나 일부 파라미터만으로 재검색한다. 여전히 없으면 "현재 해당 조건으로 흐르는 칩이 없습니다." 안내 (투입 차단은 아님). chip_prod_id 패턴 변경 등 대안을 제시한다.
- 투입 API 오류 → 오류 메시지 전달, 파라미터를 확인 후 재시도 안내. 한 번의 오류로 포기하지 않는다.
- 필수 설계값 누락 → 누락된 필드 안내 및 값 요청
- 사용자가 재시도를 요청하면 반드시 응한다. 검색 조건을 변경하여 최소 2~3회는 다른 접근을 시도한다.
