# 패턴: 자율 반복 / 비교 / 추천

사용자가 "알아서 여러 번 돌려보고 제일 좋은 거 찾아줘" 류의 복합 요청을 하면 이 패턴을 적용한다. 모델이 직접 탐색 전략을 세우고, tool을 반복 호출하며, 결과를 비교해 최적 조건을 추천한다.

## 핵심 원칙

이 패턴이 작동하려면 모델이 **자유롭게 판단**해야 한다. 사용자의 최종 목표를 이해하고, 어떤 tool을 어떤 순서로 몇 번 호출할지 스스로 결정한다. 아래는 가이드일 뿐 엄격한 순서가 아니다.

## 작동 흐름

### 1. 목표 파악

사용자의 요청에서 최종 목표를 추출한다.

- "신뢰성 제일 좋은 설계" → 목표: reliability_pass_rate 최대화
- "용량 제일 높게 나오는 설계" → 목표: predicted capacity 최대화
- "신뢰성 통과하면서 용량 최대" → 목표: reliability 기준 충족 + capacity 최대화
- "두께를 최대한 줄이면서 용량은 유지" → 목표: thickness 최소화 under capacity 제약

### 2. 탐색 전략 수립 및 사용자 공유

전략을 세우고 사용자에게 간단히 알린다. 이 단계는 사용자가 방향을 수정할 기회를 주기 위한 것이다.

예:
```
탐색 전략:
1. 먼저 optimal_design으로 넓게 DOE를 돌려 top 5를 구합니다
2. top 5 각각에 reliability_simulation을 돌립니다
3. 신뢰성 80% 이상인 후보 중 용량이 가장 높은 설계를 추천합니다

이대로 진행할까요?
```

사용자가 OK하면 진행한다. 전략 변경 요청이 오면 반영한다.

### 3. Tool 반복 호출

전략에 따라 tool을 호출한다. 대표적인 조합 패턴:

**패턴 A: 최적설계 → 신뢰성 검증**
```
optimal_design(...) → top 5
for each candidate in top 5:
    reliability_simulation(candidate.design)
→ 신뢰성 기준 통과한 후보 중 최적 추천
```

**패턴 B: 파라미터 스윕으로 신뢰성 탐색**
```
핵심 파라미터 1~2개를 축으로 선정 (예: Cover T, Margin L)
for each value combination:
    reliability_simulation(...)
→ 통과확률이 가장 높은 조합 추천
```

**패턴 C: 신뢰성 필터 → 최적설계**
```
여러 설계 조건으로 reliability_simulation 반복
→ 통과확률 기준 충족하는 조건 확인
→ 그 조건을 params 범위로 제한해서 optimal_design
→ top 5 중 목표에 가장 부합하는 후보 추천
```

**패턴 D: 반복 최적화**
```
optimal_design → top 1 후보 선택
→ 해당 후보에서 핵심 파라미터 미세 조정
→ 재실행 (params 단일값)
→ 개선되면 계속, 아니면 다른 축 시도
→ 수렴하면 최종 추천
```

### 4. 중간 결과 보고

tool 호출이 많을 때는 중간중간 진행 상황을 요약해서 보여준다.

예:
```
[3/5 완료] 지금까지 결과:
- 후보 1: 통과확률 0.87, 용량 10.12uF ✅
- 후보 2: 통과확률 0.91, 용량 10.05uF ✅
- 후보 3: 통과확률 0.72, 용량 10.21uF ❌ (기준 미달)
나머지 2개 진행 중...
```

### 5. 최종 추천

모든 결과를 모아서 비교 테이블로 정리하고, 최종 추천을 이유와 함께 제시한다.

```
종합 비교:

| 후보 | Sheet T | Cover T | 전극수 | 예측용량 | 통과확률 | 공정표준 |
|---|---|---|---|---|---|---|
| 2 | 5.2 | 30 | 158 | 10.05uF | 0.91 | ✅ |
| 1 | 5.0 | 30 | 162 | 10.12uF | 0.87 | ✅ |
| 4 | 5.1 | 30 | 160 | 9.94uF | 0.85 | ✅ |

추천: 후보 2
- 신뢰성 통과확률이 91%로 가장 높음
- 용량도 target 10uF에 +0.05uF로 충분
- 공정검사표준 전 항목 부합
```

## 제약 사항

- 한 번에 10회 이상 tool 호출이 필요한 탐색이면 사용자에게 범위를 먼저 확인한다
- 탐색 도중 사용자가 방향을 바꾸면 즉시 반영한다
- 최종 추천에는 반드시 근거를 포함한다 (어떤 조건에서 어떤 결과가 나왔는지)
- 공정검사표준 검증은 최종 추천 후보에 대해 반드시 수행한다

## 사용자 요청 예시와 전략 매핑

| 사용자 요청 | 조합 패턴 | 예상 tool 호출 수 |
|---|---|---|
| "알아서 신뢰성 좋은 설계 찾아줘" | B (스윕) 또는 A (DOE→신뢰성) | 5~10회 |
| "신뢰성 좋은 것 중에 용량 최대" | A (DOE→신뢰성 필터→용량 정렬) | 6~8회 |
| "여러 조건 바꿔보면서 비교해줘" | B (스윕) | 3~8회 |
| "후보 3개 각각 신뢰성 비교" | 단순 반복 | 3회 |
| "최적설계 돌리고 제일 좋은거 신뢰성까지 확인" | A (DOE→신뢰성) | 6회 |
